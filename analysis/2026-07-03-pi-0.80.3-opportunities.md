## 1. Changelog capabilities in range (cited)

### 0.79.9 (Jun 20, 2026)
- **Chat-template thinking compatibility** — OpenAI-compatible custom providers can integrate pi thinking levels into `chat_template_kwargs` (vLLM/HF models like DeepSeek).
- **GLM-5.2 provider improvements** — Fireworks OpenAI-compatible routing corrected; OpenRouter `xhigh` thinking support.

### 0.79.10 (Jun 22, 2026)
- **Extension compaction event context** — `session_before_compact` / `session_compact` extension events now carry `reason` and `willRetry` metadata, distinguishing manual `/compact`, threshold auto-compaction, and overflow-retry flows.
- **Safer update flow** — `pi update` installs the exact checked version, `--force` reinstalls, fails rather than falling back on unavailable versions, and reports both old and new version numbers.

### 0.80.0 (Jun 23, 2026)
- **pi-ai global API relocated to `/compat`** — legacy global methods (`stream`/`complete`/`completeSimple`, `getModel`/`getModels`/`getProviders`, `registerApiProvider`, `getEnvApiKey`) moved from the `@earendil-works/pi-ai` root to `@earendil-works/pi-ai/compat`. Release note verbatim: "Extensions continue functioning without changes due to loader resolution, but source code should migrate imports to the compat path or adopt the new `createModels()`/provider-factory API."
- **Removed** the selective-provider entrypoints `@earendil-works/pi-ai/base` and `@earendil-works/pi-agent-core/base` (added in 0.79.8; removed one minor later).
- **Fix (extension DX):** extension crash and startup failure reporting improved, with a `pi -ne` restart suggestion.
- `Ctrl+J` newline keybinding; `zai` label rename; session-name newline normalization; session-selector ordering; various provider fixes.

### 0.80.1 (Jun 23, 2026)
Three provider fixes only (Bedrock `AWS_PROFILE` endpoint resolution, Fireworks Anthropic-compatible session affinity, Together MiniMax M2.7 metadata).

### 0.80.2 (Jun 23, 2026)
- **`ApiKeyCredential` discriminator changed** to `type: "api_key"` (from `type: "api-key"`) with provider-scoped `env` values, for `auth.json` compatibility.
- **Type rename:** agent-core `ExecutionEnvExecOptions` → `ShellExecOptions`.
- **Fix:** legacy per-API stream aliases (e.g. `streamSimpleOpenAICompletions`) restored on the pi-ai `/compat` entrypoint.

### 0.80.3 (Jun 30, 2026)
- **Claude Sonnet 5 support** — via inherited Anthropic-compatible and Bedrock provider catalogs, **with adaptive thinking enabled**.
- **`Usage.reasoning` token counts** — for providers reporting reasoning/thinking tokens.
- **Richer RPC session access** — `get_entries` and `get_tree` RPC commands for inspecting session entries and tree snapshots; new **`./rpc-entry` package export for launching pi in RPC mode**.
- **`session_info_changed` extension event** — extensions can observe session name changes.
- **`outputPad`** setting (message/thinking padding); **`externalEditor`** setting (Ctrl+G); Azure Foundry endpoint support; default OpenAI model → `gpt-5.5`.
- **Behavior change:** OpenAI Codex Responses SSE header waits use the configured HTTP timeout instead of a fixed 20s timeout.
- **Notable fixes (as listed):** "deterministic session IDs for ephemeral runs", "extension tool changes timing", "provider stream error auto-retry", "streamSimple() max-token caps", "pre-prompt compaction flow", among others.

## 4. Implementation-shape impacts on planned work

- **normalizeToolChoice shim stays.** No item in 0.79.9–0.80.3 mentions `toolChoice` honoring on the `openai-responses` family or google drivers — the limitation the shim exists for (`jit-runtime.ts:294-311`) is not addressed. The shim does not retire. One caveat: the shim's doc comment pins its driver-behavior claims to "pi-ai 0.70.2" (jit-runtime.ts:294); nine-plus minors of drift sit between that anchor and 0.80.3's pi-ai. The changelog gives no signal of change, but the claim should be re-verified against the 0.80.x driver source when the update lands, and the comment's version anchor refreshed.
- **FGAP-104 fix shape unchanged.** No release in range touches skills tooling, skill generation, or extension-factory execution semantics. The hermetic + fail-loud resolution filed in FGAP-104 proceeds as filed.
- **FGAP-102, TASK-022, TASK-057, TASK-058, and the nine milestones: no changelog capability applies.** These are pi-context-internal (catalog parity, truncation-cap measurement, schema-history audit, migration-chain walk, substrate convergence/integrity/enforcement). The pi runtime changes in range don't intersect their implementation. Stated as a null, not padded.
- **jit-runtime `complete` import migrates at update time.** The 0.79.8 → 0.80.x adoption carries one source obligation (jit-runtime.ts:22 → `/compat` or `createModels()`); runtime behavior is covered by loader resolution in the interim, and 0.80.2 restored the legacy per-API aliases on `/compat`, so the migration is low-risk either way.
- **Sonnet 5 pinning is gated on the adaptive-thinking × forced-toolChoice check** (§3 row 1) for the classify path specifically; ordinary workflow-step `--models` pins have no such interaction.
