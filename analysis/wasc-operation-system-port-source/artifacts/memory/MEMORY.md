# Memory index

- [Directive states outcome, not fixture construction](feedback-directive-states-outcome-not-fixture-construction.md) — IMPL/agent directives + criteria state what must be TRUE, never how to build the test/fixture; if you do prescribe a fixture, trace it against every whole-draft invariant/consumer (e.g. coverage) before shipping; an agent's self-justified reinterpretation isn't validation — the separate adversarial audit is (TASK-052: over-specified directive's internal contradiction forced an IMPL deviation)

- [Only act on explicit directives](feedback-only-act-on-explicit-directives.md) — never infer/propose/AskUserQuestion an action from a statement of intent, observation, rationale, or "my revisions are to…"; that is CONTEXT, not a directive; report/establish only what was asked, then stop
- [Don't punt researched decisions as questions](feedback-dont-punt-researched-decisions-as-questions.md) — when research/investigation was tasked to determine an answer, DECIDE it with the evidence; never AskUserQuestion a technically-determinable choice back to the user (they delegated it because they're not the source); reserve questions for genuine preference/scope/priority only

- [Iterate-to-zero, no pressure deviation](feedback-iterate-to-zero-no-pressure-deviation.md) — an in-loop audit finding = loop the ROOT fix (re-IMPL→re-audit) until zero, NOT a mandate-008 stop-and-ask; never offer residual/defer/proportionate options or AskUserQuestion a policy-determined fix; cost/context-spend/low-threat never caps a fix; distinct new issue → file per mandate-007

- [No resuming a quiescent agent](project-no-resume-quiescent-agent.md) — SendMessage to a finished subagent is hook-blocked; to continue, spawn a FRESH foreground Agent with a self-contained prompt carrying all on-disk state + remaining steps (encoded in CLAUDE.md canonical-pipeline step 3)

- [Run the whole-project gate](project-run-the-whole-project-gate.md) — verify with the actual gate command at full scope (mypy . / make typecheck / full pytest), never a hand-picked file subset; a return-type change ripples into TEST callers a subset mypy never sees (A3: subset mypy "clean" but mypy . was RED on test_a1_narrative.py)
- [Corroborate the consumer chain of a changed return shape](feedback-corroborate-consumer-chain-of-changed-return-shape.md) — a plan that changes what a function RETURNS (shape/type) must trace EVERY consumer (callers, client type-branches, resolve/merge, tests) before the design enters the plan; verifying the fn exists isn't enough; carry new data via a NEW optional field, not a reshape (TASK-044: "dict prefill" would've broken ai-assist.js Array.isArray + C1 tests; IMPL's STOP caught the planning failure)
- [Canonical pipeline requires the plan-mode gate](feedback-canonical-pipeline-requires-plan-mode-gate.md) — run every canonical-pipeline task IN PLAN MODE (EnterPlanMode → Explore → corroborate vs source AND git ground-truth → plan → ExitPlanMode APPROVAL → then IMPL → audit → cascade); writing a plan file ≠ the plan-mode phase; never dispatch IMPL before ExitPlanMode approval; skipping it left an interrupted IMPL's unverified residue (caught via git status/diff; discarded with restore/rm, never git reset)

- [task_depends_on_task edge direction](project-task-depends-edge-direction.md) — file parent=PREREQUISITE (earlier) → child=DEPENDENT (blocked), opposite the relation name; context-current-state derives from this + context-validate won't catch a reversal; verify the unblocked frontier is the earliest tasks (exemplar FGAP-007 / CTX-8)
- [Terse persisted rules](feedback-terse-persisted-rules.md) — rules/notes the LLM will read (CLAUDE.md, .context conventions, memories) must be terse: state the directive + minimum specifics, the LLM infers the pattern; don't write exhaustive prose walls (forensic commits + ledger entries still warrant detail)
- [Explore must verify current source, not migrations](feedback-explore-verify-current-source-not-migrations.md) — Explore agents must confirm a model/vocab is LIVE via `grep class X` + delete-migration check + tests, not infer from a historical seed migration or .pyc; MissionArea was seeded (0011) then deleted (DEC-45/0061) — the Explore missed the deletion and drove a re-add plan the IMPL caught; spot-verify load-bearing Explore claims before IMPL
- [Catalogue-gate: never reject DB-enumerated values](project-catalogue-gate-no-db-enumerated-rejection.md) — the free-text gate (planner/_freetext_audit.py) must never reject a value enumerated in the school's DB (only genuine fabrications); fix globally = union COMPLETENESS (all enumerable DB fields, incl. value-words like "Caring") + variant-tolerant ADMISSION (hyphen/space/case, short/brand forms, possessives), not per-entity patches
- [.context is this repo's substrate](project-context-substrate-is-this-repo.md) — the .context here IS wasc's own; pi-context-cli's install path under workflowsPiExtension + the blocks' pi-flavored schema text are NOT "another project"; operate with `--cwd .` (or omit), file wasc bugs to the `issues` block, don't divert wasc records away from .context
- [Plan-file structure](feedback-plan-file-structure.md) — plan files (~/.claude/plans/*.md) follow ONE portable shape: Context (why) → **## Success Criteria** (the task's acceptance_criteria as a verifiable checklist, right after Context) → decisive no-forks fix (resolve forks by reading source for exact anchors), file:line anchors + reused symbols, mirror named patterns, verification = tests+runtime-demo+fresh-adversarial-audit-loop-to-zero (proves each success criterion), a Discipline section, Critical files; exemplar iridescent-nibbling-wand.md

- [Flagging is not persistence](feedback-flagging-is-not-persistence.md) — "I'll note/flag so it isn't lost" preserves nothing; write to a durable doc (DEC/DISC/US/LOG/STATE/memory) that turn, prefer a first-class entry over a buried paragraph; never say "flagged so it isn't lost"

- [User decision is a directive to act](feedback-user-decision-is-a-directive-to-act.md) — execute the directive given (e.g. "plan mode the onboarding"); never invent blockers/concerns about adjacent unbuilt things; a user's clarifying premise dismisses your concern, it is not a new fork to litigate/defer/decide

- [No augmenting user stories](feedback-no-augmenting-user-stories.md) — generate stories/requirements ONLY from stated/entailed intent; never add adjacent features (notifications, acknowledgments, etc.) from prior art or "naturalness"; not-raised-by-user ≠ deferred, it's not a requirement
- [Cost is not disqualifying](feedback-cost-is-not-disqualifying.md) — implementation cost/size/effort is never a counterweight to the right design; "heaviest lift"/"the cost is..." hedging is laziness; don't attribute your generated framing to the user

- [Scope the noun they named](feedback-scope-the-noun-they-named.md) — when user names a scope ("code", "this file", "the spec"), act ONLY within that noun; do not extrapolate to adjacent scopes; user shouldn't have to repeat

- [One Bash call per turn](feedback-one-bash-call-per-turn.md) — never batch dependent/mutating Bash calls; first failure cancels the rest; one tool call, see result, next
