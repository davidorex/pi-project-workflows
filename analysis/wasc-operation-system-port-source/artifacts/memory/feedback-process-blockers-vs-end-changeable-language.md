Do not get bogged down by language trivialities that block seeing end results. Classify every finding:

**TRUE PROCESS BLOCKERS** (hard-gate; full pipeline rigor; iterate-to-zero) — anything that stops a *valid end result*: the workflow/sequence doesn't run or complete; output doesn't PARSE; FABRICATION / catalogue-gate failure (names non-existent entities); STRUCTURAL gaps (uncovered learner-outcome/area with no gap surfaced, ownership not distributed, a required section missing/empty); project gate red (mypy/tests) or substrate PARITY broken.

**END-CHANGEABLE LANGUAGE** (WARN; surfaced; fixed in ONE final copy-edit pass; NEVER blocks; NO multi-agent audit loops): register/voice, coined framework/process terms, scaffolding labels, strand-prefix casing, wording/diction.

Verification tools (e.g. check_draft) hard-gate only on blockers; language is WARN. Run language tasks lightly (land the additive rule + a WARN check); reserve the full IMPL→adversarial-audit→iterate-to-zero for structural/process tasks and one final whole-draft language polish.
