This `process-running/` directory holds the remaining source-project content related to the RUNNING OF THE PROCESS that the prior bundle did not capture, persisted BYTE-VERBATIM (`cp`/`rsync -a`), readable layout preserved.

## Persistence method

- Files copied with `cp -p` / `rsync -a` (no transcription). Regenerated bytecode caches (`__pycache__/`) were not copied; everything else under each captured directory was.
- Byte-verbatim fidelity verified with `cmp -s` on every copied file: **1648 files compared directly against source, 0 mismatch, 0 missing** (1629 recursively-copied + 19 root/gate, plus the 3 app-orchestration-commands and 5 structured-JSON canonical sources independently re-`cmp`'d OK).
- Governing-decision verbatim md extracts (`governing-decision-texts/*.verbatim.md`): each entry is the exact `raw` (or `body`) string from its source JSON. Verified programmatically — every extracted string is a verbatim substring of its source JSON (0 not-found across 182 entries). The structured JSON these are drawn from is ALSO persisted byte-verbatim under `context-migration/decomposed/`, so the md is a readability layer over an already-verbatim source.
