# pi-workflows — findings ledger (issues / frictions / bugs)

**Purpose.** A running, dated ledger of every issue, friction, or bug found in
`@davidorex/pi-workflows` (and adjacent `pi-context-cli` / `.context`) while
building the workshop-vivification runner here — so this project's work can
inform pi-workflows work upstream. MD is the deliberate home: these are
*upstream/external* observations, not wasc `.context` items. Detailed fix
proposals live in `PI-WORKFLOWS-UPSTREAM-IDEAS.md`; this is the observation log
that points at them. Add a row whenever a new finding surfaces.

**Status legend:** `open-upstream` (real, unaddressed in pi-workflows) ·
`worked-around` (we route around it; no pi-workflows change needed for us) ·
`documented` (recorded as a runner-doc warning) · `closed-our-side` (resolved in
our spec/runner) · `investigated-clear` (checked, not actually an issue).
**Kind:** bug · footgun (silent/surprising but "works as coded") · friction
(ergonomic/packaging) · gap (missing capability) · doc-mismatch (source/docs ≠ runtime).
**Verified by:** RV = `WORKSHOPPING-WORKFLOW-RUNTIME-VERIFICATION.md` (ran the
engine); F1-audit = FEAT-001 adversarial audit; explore = FEAT-001 explore pass.
