# Simulation runs — iteration log

Tracked records of each process-simulation pass (sim-land; per `../SIMULATION-PROCESS-ARC.md`). Each run
captures the **exact inputs, the orchestration process, the raw sub-agent results, the synthesis, and the
process readout** — so we can track how the planning *process* evolves and let prior iterations inform the next.
These are sim-land records, NOT filings; when a readout's findings are settled they convert to FGAP/DEC/R-0009.

## Format (one file per run: `run-NNN.md`)
1. **Metadata** — id, date, route, status.
2. **Inputs** — seed, frame (ref), skeleton, divisions, scope decisions, baselines.
3. **Process** — the exact steps executed (grounding extraction, sub-agent dispatch + briefs + agent IDs, synthesis).
4. **Results** — the raw per-division sub-plans (verbatim) + the synthesis.
5. **Readout** — what worked, what the process surfaced (candidate findings for filing).
6. **Seeds-next** — what this run informs.
